# Модуль «Заказы и Замеры»

## Контекст

В систему добавляется новый модуль, по объёму сопоставимый с уже существующим модулем «Рекламации». Цель — закрыть полный цикл: расчёт КП у менеджера → создание Заказа → Заявка на замер → выезд СМ → Замер → обработка → производство → отгрузка → монтаж. Параллельно — деление по салонам внутри города, наработки менеджера с напоминаниями, автоматические рекомендации по проёмам, генерация подписного бланка замера, связка с уже существующими Рекламациями.

Источник ТЗ: `/Users/home/Downloads/Заказы и Замеры (3).xlsx` (8 листов).

## Бюджет

**Итого: 140 000 ₽**, разбито на 6 фаз. Каждая фаза — самостоятельный релиз: применяется на dev-стенд, заказчик проверяет, после приёмки — выставление счёта.

| № | Фаза | Сумма |
|---|------|-------|
| 1 | Салоны + модель Заказа + CRUD + список + фильтры | 30 000 ₽ |
| 2 | Парсинг КП (PDF) + Наработки + Заявка на замер | 30 000 ₽ |
| 3 | Замер: форма СМ + автоматические рекомендации + валидации | 25 000 ₽ |

| 6 | Перерисовка главного экрана + связка с Рекламациями | 20 000 ₽ |

## Фаза 3 — Замер: форма СМ + рекомендации (25 000 ₽)

Самая объёмная фаза по содержанию.

**Backend:**
- `orders/models.py`:
  - `Measurement(request: OneToOne→MeasurementRequest, service_manager: FK→User, measurement_date, signature_photo: FileField (nullable), created_at, updated_at)`.
  - `MeasurementOpening`: `measurement: FK→Measurement`, `order_item: FK→OrderItem (null)`, `opening_number`, `actual_height`, `actual_width`, `actual_depth`, `door_height_by_order`, `door_width_by_order`, `recommended_door_height` (auto), `recommended_door_width` (auto), `recommended_opening_height` (auto), `recommended_opening_width` (auto), `change_target` (`door`/`opening`/`both`), `new_door_height`, `new_door_width`, `opening_type` (enum `A`/`B`/`B_INVERSO`/`C`/`D`/`D_INVERSO`), `addon_width`, `face_trim_qty`, `face_trim_comment`, `back_trim_qty`, `back_trim_comment`, `extra_hardware`, `threshold`, `notes`.
  - `MeasurementAttachment` — фото/документы по проёму или ко всему замеру.
- В `Order.status` добавить `measurement_scheduled`, `measurement_done`, `measurement_processed`.
- `orders/recommendations.py`:
  - `calculate_door_recommendation(opening_h, opening_w)` → `(opening_h - 70, opening_w - 100)`.
  - `calculate_opening_recommendation(door_h, door_w)` → `(door_h + 70, door_w + 100)`.
  - `build_recommendation_text(opening_h, opening_w, door_h, door_w)` — текст по правилам листа «рекомендации» (60/80/90/105 мм пороги).
  - `validate_lift_required(openings)` — если есть проём с высотой > 2300 → `lift_available` обязателен.
  - `validate_inverso_warning(opening_type)` — флаг для красного «увеличьте высоту полотна на 1 см».
- `MeasurementViewSet`: create/retrieve/update; СМ может назначить дату через action `schedule(date)` (статус `measurement_scheduled`); отметить выполненным через action `mark_done` (статус `measurement_done`); менеджер отмечает «обработан» через action `mark_processed` (статус `measurement_processed`).

**Frontend:**
- `frontend/src/pages/Measurements/MeasurementList.tsx` — список замеров для СМ (фильтры: ожидают назначения / запланированные / сегодня / выполненные).
- `frontend/src/pages/Measurements/ScheduleMeasurement.tsx` — форма проставления даты/времени.
- `frontend/src/pages/Measurements/MeasurementForm.tsx` — мобильно-ориентированная форма с секцией на каждый проём, автоматический расчёт рекомендаций (на клиенте и на сервере), валидация лифта, красное предупреждение для Inverso.
- В `OrderDetail.tsx` блок данных замера (read-only после `measurement_done`), кнопка «Замер обработан» для менеджера.
- Маршруты `/measurements`, `/measurements/:id`.
- Пункт «Замеры» в `Navigation.tsx` для СМ/admin/leader.

**Verification:**
- Из заявки → назначить дату → создать замер → заполнить проёмы.
- Высота > 2300 → поле «лифт» становится обязательным.
- Открывание `B_INVERSO` → красное предупреждение.
- Фактические размеры → автоматический пересчёт рекомендаций (проверить пороги 60/80/90/105 мм).
- Менеджер отмечает «Замер обработан» → статус меняется.

## Фаза 5 — Workflow статусов + SMS + кроны (20 000 ₽)

**Backend:**
- `Order.status` enum дополнить: `paid`, `in_production`, `on_warehouse`, `shipped`, `completed`, `cancelled`, плюс просрочки `measurement_not_planned`, `measurement_not_done`, `measurement_not_processed`.
- `OrderStatusHistory` — лог изменений статуса (как у рекламаций).
- В `Order` методы переходов: `mark_paid`, `mark_in_production(deadline)`, `mark_on_warehouse`, `mark_shipped`, `mark_completed`, `mark_cancelled`. Каждый пишет в историю и шлёт уведомления.
- Cron-команды (Django management commands; запускаются через системный cron):
  - `check_measurement_not_planned` — если в `measurement_requested` > 1 раб. дня → `measurement_not_planned` + push СМ.
  - `check_measurement_not_done` — если день назначения прошёл → `measurement_not_done` + push.
  - `check_measurement_not_processed` — > 2 раб. дн. с `measurement_done` → `measurement_not_processed` + push менеджеру.
  - `check_action_reminders` — за указанную дату напомнить менеджеру (push + ЛК).
- `orders/sms_templates.py`:
  - При создании заявки: «Вам позвонит {имя СМ} {телефон СМ} по замеру в течение 2 раб. дн.»
  - При назначении: «Замер назначен на {дата и время}»
  - При перепланировке: то же с новой датой
  - При выполнении: «Замер выполнен. Скачать {ссылка PDF}»
  - При недозвоне: «Мы не дозвонились по замеру. Перезвоните пожалуйста {имя, телефон СМ}»
- `MeasurementViewSet` action `notify_client_call_failed` (кнопка «Отправить уведомление» / «Повторно отправить»).

**Frontend:**
- В `OrderDetail.tsx` кнопки переходов статусов по роли и текущему статусу.
- В карточке замера кнопка «Отправить уведомление клиенту» / «Повторно отправить».
- Цветовая разметка статусов в списках (просрочки красным).

**Verification:**
- Полный сценарий: заявка → SMS клиенту/контактному → СМ назначает дату → SMS контактному с датой → выполняет замер → SMS клиенту со ссылкой на PDF → менеджер обрабатывает → меняет статусы.
- Подменить даты в БД → запустить cron-команды вручную → убедиться, что статусы переходят в просроченные и push приходит.
